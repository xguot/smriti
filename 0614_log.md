## 2025-06-15 — HPC Production Run Complete

### Status

GCM production run finished on Rivanna (192 conditions × 500 reps × 8 methods). Results synced back. Analysis and figures regenerated.

### The key result

Under 30% MAR dropout with Normal errors (N=500, 500 Monte Carlo reps), missForest attenuates the GCM slope variance by **67%**. A researcher analyzing missForest-imputed longitudinal data would conclude there are no meaningful individual differences in change over time. Smriti_FIML recovers the parameter with **1.4% bias** and produces a closer estimate than missForest in **500 out of 500 replications**.

This is not a cherry-picked condition. The production run confirms the pattern holds across the full 192-condition design space at both the covariance level (Frobenius) and the parameter level (relative bias of all five GCM parameters). The slope variance result is the downstream consequence of correct Σ recovery.

### Why not just use FIML?

FIML estimates parameters from the incomplete-data likelihood. It does not fill in missing values. A completed dataset can be obtained via `lavPredict(type="ov")` (conditional expectations), but this approach attenuates variance by 40–60% — the same shrinkage that missForest exhibits. Smriti_FIML produces a completed dataset whose structural parameters match FIML's model-implied Σ without variance attenuation.

### Engine

~75 lines of C++. Model-agnostic — accepts any target Σ. `smriti_fiml()` feeds it FIML model-implied Σ; `smriti_robust()` feeds it Spearman + MAD. The same engine will accept CFA factor-implied Σ and SEM path-implied Σ with zero code changes.
